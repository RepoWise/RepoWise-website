Abstract

RepoWise is an AI-powered documentation assistant that helps developers and stakeholders quickly understand open-source software projects through natural language queries. Using advanced Retrieval-Augmented Generation (RAG) techniques and intelligent query classification, RepoWise provides accurate, context-aware answers by analyzing project documentation, commit history, and issue tracking data.

The system automatically extracts and indexes governance documents, contribution guidelines, commit data, and issue reports from GitHub repositories. When users ask questions, RepoWise intelligently routes queries to appropriate data sources and generates comprehensive answers with source attribution, making OSS exploration efficient and accessible.

Core Components

  1. GitHub Crawler: Automatically extracts documentation, commits, and issues from repositories
  2. Intent Classifier: Routes queries to appropriate data sources (documents, commits, or issues)
  3. Multi-Modal RAG Engine: Combines vector search with structured data for comprehensive answers
  4. LLM Integration: Supports local (Ollama) and cloud LLM providers
  5. Web Interface: Modern React frontend with light/dark theme

Intent Classification System

RepoWise employs a sophisticated intent classification system that automatically determines whether a query should be answered using documentation, commit history, or issue tracking data.

Figure 3: Intent classification workflow

Classification Categories

  • Document Queries: Questions about governance, contribution guidelines, project policies
    • Example: "Who maintains this project?"
    • Example: "What are the contribution guidelines?"
  • Commit Queries: Questions about code changes, contributors, development activity
    • Example: "Who are the most active contributors?"
    • Example: "What was changed in the last release?"
  • Issue Queries: Questions about bugs, feature requests, project health
    • Example: "How many open issues are there?"
    • Example: "What are the most common bugs?"

Technology Stack

Backend

  • Framework: FastAPI (Python)
  • Vector Database: ChromaDB
  • Embeddings: HuggingFace sentence-transformers
  • LLM: Ollama (llama3.2) / Cloud providers
  • Data Processing: pandas, BeautifulSoup4

Frontend

  • Framework: React 18 with Vite
  • UI Library: Tailwind CSS
  • Animations: Framer Motion
  • State Management: TanStack Query
  • Icons: Lucide React

Installation & Setup

Backend Setup

# Clone the repository
git clone https://github.com/RepoWise/backend.git
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your GitHub token and settings

# Install Ollama (for local LLM)
curl -fsSL https://ollama.com/install.sh | sh
ollama pull llama3.2:1b

# Start the server
./start_dev.sh

Frontend Setup

# Clone the repository
git clone https://github.com/RepoWise/frontend.git
cd frontend

# Install dependencies
npm install

# Configure environment
cp .env.example .env
# Edit .env with backend URL

# Start development server
npm run dev

Usage Example

Adding a Repository

POST /api/projects/add
{
  "github_url": "https://github.com/facebook/react"
}

Querying Documentation

POST /api/query
{
  "project_id": "facebook-react",
  "query": "What are the contribution guidelines?"
}

Response Format

{
  "answer": "To contribute to React, you should...",
  "sources": [
    {
      "file_path": "CONTRIBUTING.md",
      "score": 0.89,
      "content": "..."
    }
  ],
  "suggested_questions": [
    "How do I submit a pull request?",
    "What is the code review process?"
  ]
}

Research & Publications

RepoWise builds upon research in retrieval-augmented generation, intent classification, and open-source software analysis. Our work demonstrates that combining multiple data sources (documentation, commits, issues) with intelligent routing significantly improves the quality and relevance of answers compared to single-source approaches.

Key Research Contributions

  • Novel intent classification system for OSS queries
  • Multi-modal RAG architecture combining structured and unstructured data
  • Efficient two-stage search using Matryoshka Representation Learning
  • Comprehensive evaluation on real-world OSS repositories
